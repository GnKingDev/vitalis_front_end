# Guide utilisateur Vitalis – Index

Ce dossier contient le **guide utilisateur** de la plateforme Vitalis, rédigé en Markdown. Il peut être fourni à un outil ou une IA pour générer un document conçu (PDF, mise en page, etc.).

## Fichiers du guide

| Fichier | Contenu |
|---------|--------|
| **00-INTRODUCTION.md** | Vue d'ensemble, rôles, structure du guide, parcours patient |
| **01-CONNEXION-ET-DASHBOARD.md** | Connexion, changement de mot de passe, tableau de bord |
| **02-ACCUEIL-RECEPTION.md** | Liste des patients, enregistrement patient, tous les paiements, paiement labo/imagerie, assignation médecin |
| **03-MEDECIN.md** | Patients assignés, consultation (tous onglets), demandes labo/imagerie, résultats, dossiers archivés |
| **04-LABORATOIRE-ET-IMAGERIE.md** | Demandes laboratoire, demandes imagerie, saisie résultats, validation, PDF |
| **05-PHARMACIE.md** | Stock produits, alertes, catégories, paiements pharmacie, ordonnances |
| **06-ADMINISTRATION.md** | Utilisateurs, statistiques, tests, lits, prix consultation, sociétés d'assurance, numéros lab |

## Ordre de lecture recommandé

1. **00-INTRODUCTION.md** – Pour le contexte et les rôles.
2. Puis selon le rôle : **01** (tous) → **02** (accueil) ou **03** (médecin) ou **04** (labo) ou **05** (pharmacie) ou **06** (admin).

## Note pour la conception du document

- Les titres de niveau 1 (`#`) sont les chapitres principaux.
- Les titres de niveau 2 (`##`) sont les sections (écrans ou groupes d’actions).
- Les titres de niveau 3 (`###`) détaillent les sous-parties (objectif, filtres, actions).
- **Menu**, **URL**, **Bouton**, **Modal**, **Filtre** sont des termes récurrents à mettre en évidence si besoin.
- Les tableaux Markdown peuvent être convertis en tableaux mis en forme dans le document final.
